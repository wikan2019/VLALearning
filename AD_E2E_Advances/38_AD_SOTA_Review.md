## 2. 性能基准对比

### (a) nuScenes Planning Benchmark

开环规划评估：模型输出未来轨迹，与真实轨迹对比。

| 模型 | L2 (1s) ↓ | L2 (2s) ↓ | L2 (3s) ↓ | L2 (avg) ↓ | Col. Rate (avg) ↓ |
|------|-----------|-----------|-----------|------------|-------------------|
| ST-P3 (2022) | 1.33 | 2.11 | 2.90 | 2.11 | 0.23% |
| UniAD (2023) | 0.48 | 0.96 | 1.65 | 1.03 | 0.05% |
| VAD-Tiny (2023) | 0.46 | 0.76 | 1.12 | 0.78 | 0.22% |
| VAD-Base (2023) | 0.41 | 0.70 | 1.05 | 0.72 | 0.07% |
| GenAD (2024) | 0.36 | 0.83 | 1.55 | 0.91 | 0.04% |
| SparseDrive (2024) | 0.29 | 0.56 | 0.97 | 0.61 | 0.04% |
| DriveTransformer (2025) | **0.25** | **0.49** | **0.82** | **0.52** | **0.03%** |
| UniUGP (2025) | 0.27 | 0.51 | 0.85 | 0.54 | 0.03% |

```
nuScenes L2 Error 进展 (avg, 3s):
      ↑ L2 Error (m)
  3.0 │ ■ ST-P3
      │
  2.0 │
      │
  1.5 │       ■ UniAD
      │
  1.0 │          ■ VAD   ■ GenAD
      │
  0.5 │                         ■ DriveTF  ■ UniUGP
      │
  0.0 └──────────────────────────────────────────→ 时间
      2022      2023       2024        2025
```

> **注意**：nuScenes 开环指标的局限性在于它只衡量与 ground truth 的偏差，不评估闭环交互能力。开环 L2 低不代表真实驾驶好。

### (b) NAVSIM Benchmark

NAVSIM 提供非反应式闭环评估，用 PDM Score 综合衡量驾驶质量。

| 模型 | NAVSIM Score ↑ | No at-fault Collision ↑ | Comfort ↑ |
|------|---------------|------------------------|-----------|
| PDM-Open (Rule) | 74.4 | 92.0 | 99.8 |
| UniAD (2023) | 36.9 | 78.4 | 96.7 |
| VAD (2023) | 48.0 | 82.5 | 97.4 |
| TransFuser (2024) | 52.1 | 84.0 | 97.8 |
| EMMA-nano (2024) | 72.2 | 89.6 | 99.2 |
| DriveTransformer (2025) | 78.5 | 93.3 | 99.5 |
| Hydra-MDP (2025) | 82.1 | **94.7** | **99.7** |

> **关键发现**：在 NAVSIM 上，早期端到端模型（UniAD, VAD）甚至不如简单规则基线 (PDM-Open)——这暴露了开环训练的协变量漂移问题。2025 年的模型（DriveTransformer, Hydra-MDP）才真正超越规则基线。

### (c) 闭环评估 (CARLA / nuPlan)

| 模型 | 平台 | Driving Score ↑ | Route Completion ↑ | Infraction Score ↑ |
|------|------|----------------|--------------------|--------------------|
| Transfuser (2022) | CARLA | 61.2 | 78.4 | 0.78 |
| TCP (2023) | CARLA | 75.1 | 85.6 | 0.88 |
| ThinkTwice (2023) | CARLA | 83.2 | 90.1 | 0.92 |
| DriveVLM (2024) | CARLA | 79.5 | 88.3 | 0.90 |
| PDM-Closed (2024) | nuPlan | 92.8 | 96.5 | 0.96 |
| PlanTF (2024) | nuPlan | 90.3 | 94.2 | 0.96 |

> **CARLA vs nuPlan**：CARLA 是纯仿真环境，nuPlan 基于真实驾驶数据回放。两者结果不能直接对比，但都说明闭环评估比开环更能反映真实驾驶能力。

## 3. 架构设计对比

### 感知表示对比

| 模型 | 感知表示 | 优势 | 劣势 |
|------|---------|------|------|
| UniAD, VAD | BEV + Query | 结构化，可解释 | 需要显式 BEV 编码 |
| GenAD | BEV + 生成式 | 多模态轨迹 | 训练不稳定 |
| EMMA | 原始图像 → VLM | 端到端最彻底 | 计算量巨大 |
| DriveTransformer | Sparse Query | 高效，动态 | 稀疏可能遗漏 |
| Tesla FSD | 占据网络 + BEV | 工程成熟 | 闭源不可验证 |

### 规划方法对比

```
规划方法演进:
┌───────────────────────────────────────────────────────────┐
│                                                           │
│  (1) 回归式 (Regression)         UniAD, VAD              │
│      直接回归轨迹坐标 (x,y,t)                              │
│      简单但只能输出单一轨迹                                 │
│          ↓                                               │
│  (2) 生成式 (Generative)         GenAD, UniUGP            │
│      VAE/Diffusion 生成多条候选轨迹                         │
│      多模态输出，覆盖多种可能                               │
│          ↓                                               │
│  (3) 语言建模式 (Language Modeling)  EMMA, AutoVLA         │
│      轨迹坐标作为 token 自回归生成                           │
│      与 VLM 推理能力统一                                    │
│          ↓                                               │
│  (4) 统一生成式 (Unified Generation) DriveTransformer      │
│      感知-预测-规划全部统一为 token 生成                      │
│      最彻底的端到端                                        │
│                                                           │
└───────────────────────────────────────────────────────────┘
```

### 训练范式对比

| 模型 | 训练方法 | 闭环训练 | 数据规模 |
|------|---------|---------|---------|
| UniAD | IL (Imitation Learning) | ❌ | nuScenes (34h) |
| VAD | IL | ❌ | nuScenes (34h) |
| GenAD | IL + VAE | ❌ | nuScenes (34h) |
| EMMA | IL (VLM fine-tune) | ❌ | Waymo 内部 (大规模) |
| DriveTransformer | IL | ❌ | nuScenes (34h) |
| Tesla FSD v13 | IL + RL (闭环) | ✅ | **70 亿+ 英里** |
| Waymo FM | IL + RL + Sim | ✅ | **亿级英里 + 百亿仿真** |

> **关键差距**：学术模型主要在 nuScenes 的 34 小时数据上训练，而工业系统使用的数据量大 **5-6 个数量级**。这是学术与工业之间最根本的鸿沟。

### (d) 学术研究 vs 工业部署的鸿沟

| 维度 | 学术研究 | 工业部署 |
|------|---------|---------|
| 数据规模 | 34h (nuScenes) | 数十亿英里 |
| 评估方式 | 开环 L2 / NAVSIM | 闭环实车 + 百万用户 |
| 训练方式 | IL only | IL + RL + 闭环 |
| 传感器 | 通常仅 Camera | Camera + LiDAR + Radar |
| 安全验证 | 基准分数 | 多层安全体系 |
| 计算资源 | 8-64 GPU | 数万 GPU 集群 |
| 迭代周期 | 论文发表周期 | 每周 OTA 更新 |

> **启示**：学术模型在基准上的 SOTA 并不能直接转化为工业产品。数据规模、闭环训练、安全验证三道门槛将学术与工业隔开。

## 5. 技术趋势预测 (2026-2028)

### 六大趋势

```
技术趋势 (2026-2028):
───────────────────────────────────────────────────────────────
  趋势 1: VLM 驾驶走向成熟
  ──────────────────────────
  EMMA (2024) → 更多 VLM-based 系统 → VLM 成为驾驶标准架构
  关键：推理延迟降至 <50ms，才能真正上车

  趋势 2: 世界模型成为标配
  ──────────────────────────
  GAIA-2, NWM → 世界模型用于数据增强 + 闭环训练 + 安全验证
  关键：视频生成质量 + 物理一致性

  趋势 3: RL 闭环训练成为标准
  ──────────────────────────
  Tesla (IL+RL) → AD-R1, RIFT → 学术界也开始闭环训练
  关键：仿真器保真度 + RL 稳定性

  趋势 4: 法规逐步跟上
  ──────────────────────────
  EU R2022/1426 → ISO 5083 → 各国 L3/L4 准入落地
  关键：如何验证端到端黑箱模型的安全性

  趋势 5: LiDAR vs Camera-Only 趋于收敛
  ──────────────────────────
  Tesla (纯视觉) vs Waymo (多传感器) → 混合方案
  预测：视觉为主 + 低成本 LiDAR 补充 成为主流

  趋势 6: 数据飞轮效应加剧
  ──────────────────────────
  有车队的企业 (Tesla/Waymo/中国车企) 加速领先
  纯学术研究需要找到新的数据获取途径
───────────────────────────────────────────────────────────────
```

| 选型维度 | 选择 A | 选择 B | 建议 |

## 7. 总结

```
端到端自动驾驶 SOTA 全景:

      学术前沿                    工业部署
  ┌──────────────┐          ┌──────────────┐
  │ UniAD (2023) │          │ Tesla FSD    │
  │ VAD   (2023) │  ← GAP → │ Waymo FM     │
  │ GenAD (2024) │          │ 华为 ADS 3.0  │
  │ DriveTF(2025)│          │ 小鹏/蔚来/理想 │
  └──────────────┘          └──────────────┘
        ↑                         ↑
    开源、可复现                 闭源、大数据
    开环为主                   闭环 + 实车
    34h 数据                  数十亿英里
```

**核心要点回顾**：

1. **性能趋势**：2023→2025，nuScenes L2 误差从 1.03m 降至 0.52m，但开环指标意义有限
2. **架构演进**：回归式 → 生成式 → 语言建模式 → 统一生成式，端到端程度不断加深
3. **工业主导**：Tesla 和 Waymo 在数据规模和闭环训练上有不可逾越的优势
4. **中国追赶**：华为、小鹏、蔚来在量产端到端方面进展迅速，特别是无图方案
5. **未来方向**：VLM 驾驶、世界模型、RL 闭环训练将是 2026-2028 的三大主线
6. **安全为王**：无论架构多先进，Safety Wrapper 是产品化的必要条件
